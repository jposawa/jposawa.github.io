Segundo Dojo 2017

Antes de mais nada, gostaria de agradecer a todos que participaram desse Dojo. Independente se você conseguiu ou não resolver o problema durante o evento, é interessante que dê uma olhada.

Para fins de metodologia, essa recaptulação das questões se encaixa no formato Kata da metodologia. Diferente do Kake, no qual vocês praticavam em duplas, nessa eu mostro pelo menos uma forma de resolver as questões. No entanto é encorajado que pesquisem e tentem alterar os códigos para chegar ao mesmo resultado usando caminhos diferentes

As questões 01 e 02 foram resolvidas em Scratch E JavaScript. O primeiro usei o editor online do Scratch, enquanto que para o JavaScript usei o Repl.it


Questão 01

Enunciado

Faça um programa que o Usuário informa duas notas escolares distintas. Com essas notas o programa deve dizer se, com essas notas, o aluno em questão está APROVADO, vai para AVALIAÇÃO FINAL, ou se está REPROVADO.

As regras para cada categoria são:

À partir de 7, o aluno está APROVADO

Abaixo de 7, mas até 4, vai para AVALIAÇÃO FINAL

Abaixo de 4, é automaticamente REPROVADO

Questão 02

Enunciado

Faça um programa que o Usuário insere o número da velocidade de um veículo. O programa deve informar, com base nas regras abaixo, se o motorista do veículo deve pagar uma multa. E, caso deva pagar, qual o valor.

REGRAS

Até 60km/h, não foi multado.

Até 25% acima da velocidade limite, deve-se pagar R$ 210,00

Acima de 25%, mas até 50% acima da velocidade limite, deve-se pagar um adicional de R$ 150,00 sobre o valor anterior

Acima de 50%, além do valor acumulado, deve-se pagar R$ 2,00 para cada quilômetro excedente.

Questão 03

Enunciado

Crie um jogo que dê a opção do Jogador escolher números de 1 a 3. O número escolhido leva a um prêmio específico. Os prêmios de cada opção devem ser gerados aleatoriamente antes do jogador escolher a opção que quer.

Para fins lúdicos e simplificação, recomenda-se o uso de valores numéricos para premiação. Porém, caso o resultado daquela opção seja 0, informe o texto que o jogador não ganhou nada naquela rodada


Resolução

Devido limitações técnicas do Scratch, que não possui uma maneira nativa e intuitiva de usar Switch, precisando recorrer à "Gambiarras", a resolução foi feita apenas em JavaScript

Questão 04

Enunciado

Faça um pequeno jogo de Quiz com pelo menos quatro questões. Cada questão deve possuir pelo menos três alternativas.

Para selecionar a alternativa que deseja responder, é recomendado permitir que o usuário selecione o número da alternativa da resposta.

Após cada questão, informe se o jogador acertou a resposta, e ao final indique quantas questões ele acertou do total de perguntas.


Resolução

Pelo mesmo motivo da questão anterior, a resolução foi feita apenas em JavaScript
